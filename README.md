## Methodology

The forecasting workflow follows a chronological time-series pipeline:

1. Download historical `BTC-USD` data with `yfinance`.
2. Load and clean the dataset.
3. Select the closing price as the prediction target.
4. Compute technical indicators: `SMA`, `EMA 20`, `EMA 50`, `EMA 200`, `RSI`, `MACD`, and `MACD Signal`.
5. Remove missing values generated by rolling indicators.
6. Create lagged closing-price windows using a 7-day lookback.
7. Normalize the dataset with `MinMaxScaler`.
8. Split the data chronologically into training and testing sets.
9. Train and evaluate deep learning forecasting models.
10. Compare actual vs. predicted prices using visualizations and regression metrics.

## Models

### Recurrent Model in PyTorch

A recurrent neural network based on `torch.nn.LSTM` is used to model temporal dependencies from normalized time windows.

| Parameter | Value |
| --- | --- |
| Architecture | LSTM |
| Loss function | `MSELoss` |
| Optimizer | `Adam` |
| Learning rate | `0.001` |
| Epochs | `30` |
| Batch size | `16` |
| Lookback | `7` days |

### Chronos Forecasting

The notebook also includes an additional forecasting experiment using `amazon/chronos-t5-tiny` through `ChronosPipeline`. The model generates a 31-day forecast with quantile-based prediction intervals.

### Autoformer Reference

The notebook includes a reference Autoformer training command for a 31-day prediction horizon:

```bash
python -u BTC-FORECAST.py \
  --is_training 1 \
  --root_path ./dataset/Cryto/ \
  --data_path BTC-USD.csv \
  --model_id BTC_96_31 \
  --model Autoformer \
  --data custom \
  --features M \
  --seq_len 96 \
  --label_len 48 \
  --pred_len 31 \
  --e_layers 2 \
  --d_layers 1 \
  --factor 3 \
  --enc_in 1 \
  --dec_in 1 \
  --c_out 1 \
  --des BTC_Price_Prediction \
  --itr 1 \
  --train_epochs 100
```

## Installation

Using a virtual environment is recommended.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
```

Install the main dependencies:

```bash
pip install notebook pandas numpy matplotlib seaborn scikit-learn torch transformers yfinance ta chronos-forecasting
```

If you run Chronos on GPU, make sure your PyTorch installation is compatible with your CUDA version.

## Usage

Clone the repository:

```bash
git clone https://github.com/Ivo196/Crypto-Price-Forecasting-Using-Autoformer.git
cd Crypto-Price-Forecasting-Using-Autoformer
```

Open the notebook:

```bash
jupyter notebook "Crypto price forecasting using Autoformer.ipynb"
```

Run the notebook cells in order to:

- Load or download `BTC-USD.csv`.
- Compute technical indicators.
- Prepare training and test data.
- Train the forecasting model.
- Generate predictions.
- Evaluate and visualize the results.
